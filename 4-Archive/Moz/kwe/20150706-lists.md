#### Create List
- Create entry in `keyword_lists` table using:
  - `account_id`
  - `title`
  - the rest are defaulted
- With CSV or given an optional keyword input, for newly created `keyword_list_id`:
  - Medium priority `serp_volume_collection` task for all provided keywords
  - On completion of collection task, run `list_metric_calculations`

## Tasks

#### serp_volume_collection

The steps to gather all required information for a single keyword. Information required for this step includes:
- `keyword`
- `account_id`

Steps include:

- SERP retrieval:
  - [POST /collector/:engine/:locale/:query(/:vicinity)](https://github.com/seomoz/silo/wiki/HTTP-API-Documentation%3A-Collector%3A-POST-%3Aengine-%3Alocale-%3Aquery-%3Avicinity#post-collectorenginelocalequeryvicinity)
  - ose-api/serp-complete called (with SERP in body) by SILO
  - add to `SERPS` table for user (or is it account?)
- Grepwords: Volume
  - For who/how to use, see
    - [grepwords_client](https://github.com/seomoz/grepwords_client)
    - [vanguard/bing_ads](https://github.com/seomoz/vanguard/blob/8637f893b8ccbbdfba462ec0baa8643a87c25e23/lib/models/bing_ads/keyword_volume.rb)

#### list_metric_calculations

- `select * from keyword_list_items where keyword_list_id == 'keyword_list_id'` and iterate calculating:
  - total number of expired serps
  - total number of keywords
  - number of search engines
  - volume range
  - average difficulty
  - average opportunity
- Insert/update into `list_metrics` table

#### Notes
- Long term storage is focused towards lists. This means that Social, KWD Suggestions & Mentions will be in cache, not mysql
- Should we have  Qless cleaning up orphaned or expired serps?
- Each serp needs to have metrics (i.e. opportunity & difficulty) pulled out and placed into `keyword_metrics` alongside `volume`
- the `keyword`, `engine` and `locale` are in both the `serps` and `keyword_list_items` tables because:
  - the `serp` can expire (and be deleted) but the `keyword_list_item` will always need these details
  - we want the ability to `select results from serps where keyword = kwd and locale = loc and engine = eng` since mutually exclusive lists may request the same serp
